Section 1 of 4: System Architecture & Build Order

Phase 1 — Foundation
Auth System
Google OAuth (users) · Email/bcrypt (admin) · JWT · requireAuth middleware
Subscription Engine
8-state machine · Idempotency keys · Plan config table (admin-controlled durations + unlock rules) · Soft commercial lock post-payment
Pricing Engine
Per-day calc · Mixed meal days · Admin-configurable discount table · Price versions frozen at purchase · Wallet + promo stacking at checkout
Policy Engine
Central rule layer: skip eligibility · pause eligibility · cutoff validation · refund rules · admin override permissions · plan lock rules · plan unlock rules (30-day)
Billing / Ledger
ledger_entries table · Types: payment/credit/refund/promo/skip-comp/delivery-comp/streak-reward · Wallet balance = ledger sum · Human-language descriptions
Event Bus (pg-boss)
DomainEvent enum · emitEvent() dispatch · Job handlers wired per event · Retry + backoff · Job table archiving
Phase 2 — Operations
Scheduling Engine
pg-boss cron: 11:30PM nightly · Generate meal_cells for next day · Apply pending skips · Assign delivery slots · Kitchen prep counts output
Kitchen Operations
Prep counts per meal type · Item-wise quantities · Packaging counts · Substitution rules (item unavailable) · Admin-readable daily report
Skip Engine
Routes through Policy Engine · Pre-cutoff: auto-approve → ledger credit · Post-cutoff: admin approval queue · Emergency: admin override · Day validity preserved
Delivery System
scheduled→preparing→out→delivered · Failed: auto ledger credit via pg-boss · Late: SLA breach + compensation · Bulk actions · Admin override any status
Admin Control System
Override anything via Policy Engine · Bulk actions · All sensitive actions → Audit Log · Meal load prediction · Revenue + skip dashboard
Audit Log
audit_logs table · Every sensitive admin action: who · what · when · old value → new value · Non-deletable
Phase 3 — Support & Comms
Ticketing System
Types: DELIVERY/FOOD/PAYMENT/OTHER · Priority: LOW/HIGH/CRITICAL · Auto-tag by keyword · Admin: reply / refund-to-ledger / escalate / close
Live Chat
Quick back-and-forth on active issues · 15s poll or SSE · Escalates to ticket · Linked to subscription/delivery context
Notification System
Event triggers → pg-boss jobs · Channels: in-app + email (Gmail SMTP / Resend) · User mute prefs per type · 4 types: transactional/operational/critical/retention
Phase 4 — Retention
Wallet UX
Prominent dashboard card · Credit history with human copy · Auto-apply at checkout (toggle) · "We added ₹X back because…" microcopy
Streak Ladder
Admin-configurable tiers · Reward types: wallet credit / free extra / renewal discount / mix · streak_rewards config table · pg-boss daily check job
Smart Pause & Exit Flow
Cancel → intercept → offer pause · Pause instead of cancel flow · Renewal nudges 2 days before expiry · "Going somewhere? Pause instead"
Plan Unlock System
30-day plan locked until 1 completed plan · Admin configures unlock conditions · "Best Value" badge shown after user qualifies · Renewal-only pricing tier
