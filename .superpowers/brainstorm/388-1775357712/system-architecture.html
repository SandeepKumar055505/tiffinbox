TiffinPoint — Upgraded System Architecture

8 core systems. This is a retention + operations platform, not a food app.

🔁 RETENTION ENGINE MOST CRITICAL
  • Smart Pause (instead of cancel)
  • Exit interception + discount offer
  • Meal streak system + rewards
  • Missed meal recovery → wallet
  • Daily menu preview notification
  • Meal rating loop
⚙️ SUBSCRIPTION ENGINE
  • State machine (8 states)
  • DRAFT → PENDING → ACTIVE → …
  • Idempotent checkout (no duplicates)
  • Post-payment lock (no edits)
  • Per-subscription price versioning
💰 PRICING ENGINE
  • Per-day calculation (not average)
  • Mixed meal days handled correctly
  • Admin price change → new subs only
  • Wallet credit on skip/delivery fail
  • Promo code + wallet stacking
📅 SCHEDULING ENGINE
  • Nightly job → generates daily meals
  • meal_cells rows per date × meal_type
  • Skip applied before generation
  • Delivery slot assignment
  • Kitchen prep count output
⏱️ SKIP ENGINE
  • Pre-cutoff → auto-approve + wallet
  • Post-cutoff → admin approval flow
  • Emergency skip → admin override
  • Skip value → wallet_balance
  • Day still counts for plan validity
🚚 DELIVERY SYSTEM
  • Status: scheduled→preparing→out→delivered
  • Delivery failed → auto wallet credit
  • Late delivery → compensation credit
  • SLA breach tracking
  • Bulk mark delivered (admin)
💬 SUPPORT SYSTEM
  • Ticket types: DELIVERY/FOOD/PAYMENT/OTHER
  • Priority: LOW/HIGH/CRITICAL
  • Auto-tag by keyword
  • Admin: reply / refund to wallet / escalate
  • Live update polling
⚙️ ADMIN CONTROL SYSTEM
  • Override anything (skip/price/schedule/meal)
  • Bulk actions (menu change, notifications)
  • Meal load prediction (kitchen prep)
  • Tomorrow's delivery count
  • Revenue + churn dashboard
